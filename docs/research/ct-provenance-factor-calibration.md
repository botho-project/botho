# CT Provenance-Factor Calibration: Can a Value-Free Factor Replace Hidden Cluster Wealth?

**Issue:** [#902](https://github.com/botho-project/botho/issues/902) — **decision D2** of the confidential-amounts (CT) reconciliation epic (ADR 0006 decision 1). This is the single highest-leverage decision in the epic: the factor-curve's input, **cluster wealth**, is the one quantity that genuinely goes hidden under CT (`docs/research/ct-economics-gadgets.md` §4).
**Status:** Design research + simulation. **No consensus change.** The D2 decision is surfaced for maintainer ratification and carried into the CT normative spec (the `anvil:spec` authoring #902 is parked on), exactly as the lottery (Path C) and bridge-import (ADR 0007) decisions were.
**Reproduce:**
```
cargo run -p bth-cluster-tax --features cli --bin cluster-tax-sim -- provenance-factor-sweep
```
Sim source: `cluster-tax/src/simulation/provenance_factor_sweep.rs` (deterministic; seed `0xB07A_0902_D2`; expected-value, no draws → byte-for-byte reproducible, `report_is_reproducible` test + two live runs `diff`-clean). Every table below is regenerated by that command — nothing is hand-computed. It reuses the shipped kernels: the real `ClusterFactorCurve` (the log-domain sigmoid), the real `demurrage_charge`, the real `TagVector::mix` value-weighted blend (for circulation decay), and the shared `simulation::metrics::calculate_gini`.

---

## 1. The decision, stated precisely

Botho's anti-hoarding layer maps a coin's **cluster wealth** `W(c) = Σ_utxo v·weight(c)` — a value-weighted aggregate over the coin's cluster origin — onto a 1×–6× factor via the production `ClusterFactorCurve`. That factor drives demurrage, the progressive fee, and (historically) the lottery tilt. Under ADR 0006 confidential amounts the per-UTXO `v` are Pedersen commitments, so `W(c)` is only available as a homomorphic commitment. Two resolutions (`ct-economics-gadgets.md` §4):

- **Option A** — an **octave-bucket range proof** over a *committed* cluster-wealth aggregate. The curve is already a log2/octave step structure, so proving the factor = placing committed `W(c)` in a public octave bucket = ~2 range proofs, ≤16 clusters, ~1–2 KB/spend. Exact-ish; real crypto.
- **Option B** — a **value-free provenance factor** keyed on signals that stay PUBLIC under CT (mint origin size, coin age, hop count), generalizing ADR 0007's bridge-import tagging to *domestic* coins. Zero proof, coarser. **If it holds the bars, it dissolves the last hard CT gadget** the way Path C dissolved the lottery.

**The empirical bar (the same one Path C and bridge-import cleared):** does a value-free provenance factor **preserve the redistribution (Δgini) floor AND the Sybil-resistance** the live value-weighted factor delivers? This memo builds the calibration sim to answer it, and — because the task's own framing insists — tests the **Sybil axis as the crux**: provenance signals, unlike hidden value, are attacker-*manufacturable*.

## 2. Why Sybil-resistance is the hard axis

The value-weighted factor is Sybil-resistant for a structural reason: cluster wealth is a **value aggregate that inherits through splits**. A whale cannot shrink `W(c)` by fragmenting (the pieces keep the cluster tag; the sum is conserved), and it cannot shrink it at all except by *giving value away*. Value is not a "structure the holder controls for free."

Provenance signals are different. **Age** (blocks since mint) resets to 0 on any spend — a whale churns to stay young for the price of a base fee. **Hop count** is incremented by self-spends — a whale washes to fake a high circulation count for free. These are exactly the "structure the holder controls for free" that Path C proved fatal for a value-free *lottery weight*. The open question is whether the *same trap* sinks a value-free *factor*, or whether one provenance construction escapes it.

The one that might: **ADR 0007's bridge-import pattern**, generalized. Its factor is keyed on the epoch's **public** aggregate origin size, floored at `F`, and **decays only by circulation** — mixing with genuinely background-tagged value, the real value-weighted `TagVector::mix`, in which self-spends (0 incoming background value) are a no-op. That circulation-gating is precisely what a free churn/wash cannot fake. The domestic analog exists because **mint (coinbase) reward amounts are public** (PoW-bound minting), so the mint-epoch pool wealth `Σ public coinbase in [mK,(m+1)K)` is a value-free curve input.

## 3. Method

A focused, deterministic experiment (same shape as `settlement_horizon_sweep`, `lottery_selection_sweep`, `bridge_import_sweep`), because the lever is the *factor rule*, not multi-agent flow. Four rules are scored:

| rule | factor input | CT-clean | manufacturable for free? |
|---|---|:--:|---|
| **ValueWeighted** | live `Σ v·weight` | **no** (hidden) | no (split-invariant value aggregate) — the baseline CT hides |
| **Age** | blocks since mint (idle → higher) | yes | **yes** (churn resets age) |
| **HopCount** | spends (fresh → higher) | yes | **yes** (self-hop wash) |
| **EpochOrigin** | public mint-epoch pool wealth, circulation-decayed, floored `F=1.5×` | yes | **no** (decay needs real circulation) |

Each rule is scored on four tracks: **1a faithfulness** (how coarsely the value-free signal tracks the baseline factor), **1b honest redistribution** (Δgini vs a burn baseline in an honest population — 10-year collect-and-redistribute, per-capita Path-C payout), **2 Sybil manufacturability** (can a whale churn/self-hop/split to *fake* a low factor for free?), and **3 gamed redistribution** (what Δgini survives once the whale manufactures its cheapest low factor). The population is the same tier shape as the lottery sweep (a large poor/background commerce tier + a graded wealthy idle tier spanning the real curve), extended with the age/hop/mint-origin signals each rule reads.

## 4. Results

### 4.1 Faithfulness (how coarsely each value-free signal tracks the baseline)

| tier | cluster wealth | ValueWeighted | Age | HopCount | EpochOrigin |
|------|---------------:|------:|------:|------:|------:|
| background (poor) | 1000 | 1.265x | 1.050x | 1.000x | 1.000x |
| 50k-cluster wealthy | 50000 | 3.023x | 4.750x | 6.000x | 3.023x |
| 500k-cluster wealthy | 500000 | 4.617x | 6.000x | 6.000x | 4.617x |
| 10M-cluster whale | 10000000 | 5.745x | 6.000x | 6.000x | 5.745x |

Mean absolute factor error vs the ValueWeighted baseline (lower = more faithful):

| rule | mean \|factor − baseline\| |
|------|--------------------:|
| ValueWeighted | 0.000x |
| Age | 0.312x |
| HopCount | 0.407x |
| EpochOrigin | **0.238x** |

**Reading it.** `EpochOrigin` **reproduces the baseline factor exactly on every wealthy tier** (3.023×, 4.617×, 5.745×) — because it feeds the *same real curve* the same BTH-scale input (public mint-epoch wealth ≈ live cluster wealth for a genuinely concentrated holder). It is the most faithful value-free rule (0.238× mean error, driven entirely by the background tier where the origin tag has fully circulated to 1× vs the baseline's residual 1.265×). `Age` and `HopCount` are **coarse**: with no BTH scale they flatten every wealthy tier into 6× (or 4.75×), unable to tell a 50k cluster from a 10M one.

### 4.2 Track 1b — honest redistribution (Δgini vs a burn baseline)

10-year collect-and-redistribute, per-capita (Path C uniform) payout. Initial Gini 0.7773; burn baseline Gini 0.7525. Δgini = burn − rule (higher = more equalizing).

| rule | final Gini | Δgini | recovery vs baseline | CT-clean |
|------|-----------:|------:|---------------------:|:--------:|
| ValueWeighted | 0.6508 | +0.1017 | 100.0% | **no** |
| Age | 0.6325 | +0.1201 | 118.0% | yes |
| HopCount | 0.6310 | +0.1215 | 119.5% | yes |
| EpochOrigin | 0.6507 | +0.1019 | **100.2%** | yes |

**Reading it — redistribution alone does not discriminate.** In an *honest* population every rule clears the 0.05 design floor and recovers ≥100% of the baseline Δgini. The signals are correlated: an honest hoarder is wealthy *and* old *and* unspent *and* big-origin, so any of them taxes it. Age/HopCount even over-shoot (118–120%) because they max out every wealthy tier at 6× (their coarseness cuts *for* redistribution here). **This is the trap** — measured on redistribution alone, the gameable rules look best. The discriminator is Sybil (§4.3), and its redistribution consequence (§4.4).

### 4.3 Track 2 — Sybil manufacturability (the crux)

A concentrated whale (value 100k BTH, live cluster wealth 10M → baseline **5.745×**) tries to manufacture a LOWER factor with cheap manipulations. Each cell is the whale's factor AFTER the attack; **bold** = the attack is FREE (base fees only).

| rule | honest | split→K | churn young (age→0) | self-hop wash (hops→20) | real-circulate (costs value) | cheaply gameable? |
|------|------:|------:|------:|------:|------:|:--:|
| ValueWeighted | **5.745x** | **5.745x** | **5.745x** | **5.745x** | 5.745x | no |
| Age | **3.500x** | **1.000x** | **1.000x** | **1.000x** | 1.000x | **YES — broken** |
| HopCount | **6.000x** | **5.750x** | **5.750x** | **1.000x** | 3.750x | **YES — broken** |
| EpochOrigin | **5.745x** | **5.745x** | **5.745x** | **5.745x** | 1.474x | **no** |

**Reading it — this is the whole decision.** `split`, `churn young` and `self-hop wash` are all FREE (only base fees). **ValueWeighted and EpochOrigin are unmoved by every free attack.** ValueWeighted because the live aggregate is split-invariant and can only fall by giving value away. EpochOrigin because its factor decays **only** by real circulation — the value-weighted `TagVector::mix` with 0 incoming background value is a no-op, so churn/wash/split cannot shift the origin tag. **Age and HopCount collapse to 1× under a free churn/wash** — the whale fakes a young, well-hopped coin for the price of base fees and sheds its entire factor. The only thing that lowers `EpochOrigin` is `real-circulate` (to 1.474×, its floor) — and that is **not free**: the sim drives the real blend receiving a full position's worth (1× value over 9 spends, the ADR-0007 worst-case decay depth) of *genuine background money*, which the whale must actually acquire by circulating in the domestic economy. That is not an evasion; it is the mechanism working as intended (spend it into commerce and it becomes cheap money).

### 4.4 Track 3 — redistribution when the whale GAMES the factor

The same collect-and-redistribute, but the wealthy tiers manufacture their cheapest FREE low factor (Age → churn young; HopCount → self-hop → 1×). ValueWeighted and EpochOrigin cannot be gamed for free, so the wealthy keep their honest factor.

| rule | Δgini honest | Δgini gamed | survives gaming? | CT-clean |
|------|-------------:|------------:|:--:|:--:|
| ValueWeighted | +0.1017 | +0.1017 | yes | **no** |
| Age | +0.1201 | **−0.0249** | **NO** | yes |
| HopCount | +0.1215 | **−0.0248** | **NO** | yes |
| EpochOrigin | +0.1019 | **+0.1019** | **yes** | yes |

**Reading it — the money shot.** Once the whale games the factor, `Age` and `HopCount` don't merely lose their redistribution — they go **negative** (−0.025), *worse than burning the demurrage*, because the wealthy now pay ~0 into the pool while everyone still gets the per-capita payout: the lottery mildly *regresses*. `ValueWeighted` and `EpochOrigin` are **identical honest vs gamed** (the whale has no free move), holding +0.10. The Track-1b redistribution advantage of the gameable rules was a mirage; it evaporates the instant an adversary is present, which under CT it always is.

### 4.5 CT-compatibility & leakage

| rule | CT-clean | keys on | Sybil verdict |
|------|:--:|---------|---------------|
| ValueWeighted | **no** | hidden live cluster wealth Σ v·weight | resistant, **but reads hidden value → this is what Option A must prove in ZK** |
| Age | yes | public blocks-since-mint | **BROKEN — free manufacture** |
| HopCount | yes | public spend/hop count | **BROKEN — free manufacture** |
| EpochOrigin | yes | public mint-epoch pool wealth + circulation blend | **resistant (decay is circulation-gated)** |

`EpochOrigin`'s leakage beyond ADR-0006-public is **nil**: mint reward amounts are already public (PoW-bound minting), the epoch key `m = ⌊height/K⌋` is public, and the circulation blend reads only public tags. It adds nothing to the §9 metadata-leak table — the same clean bill ADR 0007 earned at the bridge boundary.

## 5. The residual: drip-mining across epochs

`EpochOrigin` is not perfectly equal to the live aggregate, and honesty requires naming where they diverge. The value-weighted factor reads the **current** cluster sum; the epoch-origin factor reads the **mint-epoch pool** at origination, decayed by circulation. A whale that accumulates a large position by **minting slowly, spread across many epochs**, gives each epoch a small pool → a low per-mint factor → a large low-factor position the live-wealth curve would have rated high. This is exactly the ADR 0007 "spread across epochs = time-as-cost" residual, and it is bounded the same way:

- **Time-as-cost.** Diluting a 2M-BTH import to the floor took **541 separate epochs** in the bridge calibration (`bridge-import-calibration.md`); at `K = 1 day` that is 541 days. The domestic bound is identical in shape.
- **Emission already throttles it.** Domestic mint income is rate-limited by the emission schedule and PoW difficulty (block reward fixed, shared across all hashrate). Accumulating a whale-scale position by mining is *inherently* spread across a long wall-clock window regardless of the attacker's intent — the "attack" is largely forced by monetary policy, not a cheap option.
- **The floor `F` caps the payoff.** Even a fully drip-mined position bottoms at `F = 1.5×`, not 1× — a non-trivial standing toll.

This residual is real but is (i) already accepted and ratified for the structurally-identical bridge boundary (ADR 0007), and (ii) far weaker than the *free, instantaneous* age/hop manufacture that sinks the naive rules. It is the one thing a maintainer must consciously accept to choose Option B.

## 6. Recommendation

**Option B holds both bars — but ONLY with the `EpochOrigin` rule. The intuitive age/hop provenance factors FAIL the Sybil bar and must not be used.**

- **`EpochOrigin` (the ADR-0007 generalization) is the recommendation.** It is CT-clean (leakage-neutral), the **most faithful** value-free rule (0.238× mean factor error; exact on the wealthy tiers), holds the honest Δgini floor at **100.2% recovery**, and — the decisive property — is **not manufacturable for free**: split/churn/self-hop leave its factor untouched (5.745× → 5.745×), because its decay is circulation-gated and its split-resistance comes from epoch-pool sharing. Its redistribution is **gaming-invariant** (+0.1019 honest and gamed). Adopting it **dissolves the last hard CT gadget**: with the factor value-free, the CT economics reconcile with **no ZK beyond the range proofs ADR 0006 already mandates** — the demurrage gadget's one Bulletproof inequality plus nothing for the factor.

- **`Age` and `HopCount` are disqualified.** They look *best* on honest redistribution (118–120%) — the trap — but a free churn/wash collapses them to 1× (Track 2), and their gamed redistribution goes **negative** (Track 3). They are the factor-side restatement of Path C's proven-impossible value-free split-invariant *weight*: any factor keyed on a per-coin structure the holder controls for free is manufacturable.

- **Option A (octave-bucket proof over committed wealth) does NOT need to stand — but remains the clean fallback.** If a maintainer judges the **drip-mine-across-epochs residual** (§5) unacceptable for the domestic path — i.e. wants the *exact* live aggregate rather than the origin proxy, safe even against a patient slow-miner — then Option A is the mechanism that delivers it, at ~1–2 KB/spend of range proofs. That is the residual crypto #902 would then need; `EpochOrigin` is what lets #902 avoid it.

### Recommended parameters for the CT normative spec (if Option B)

- **Rule:** domestic **mint-epoch origin factor** = `max(F, ClusterFactorCurve(Σ public coinbase in [mK,(m+1)K)))`, blended toward background 1× by the coin's remaining origin-tag weight (the existing value-weighted `TagVector::mix` on spends). A mint-epoch cluster is a third origin type alongside domestic-mint and bridge-import — no new machinery beyond the tag.
- **Floor `F` = 1.5×**, inherited from ADR 0007 (#940).
- **Epoch length `K` = 17,280 blocks (1 day)**, inherited from ADR 0007 — the domestic drip-mine residual has the identical `K`-vs-time trade-off, so sharing the constant is consistent and keeps one dial.
- **Decay:** circulation-only (no time decay), via the shipped blend — sitting idle never normalizes; *using* the coin does.
- **Watch item:** the drip-mine-across-epochs residual (§5), bounded by `K` and by emission throttling. If a future high-hashrate-concentration regime makes it material, escalate to Option A for the domestic path (the bridge path already lives with the identical residual).

## 7. Bottom line for #902 (D2)

- **The value-free reframe works for the factor — with the right rule.** `EpochOrigin` preserves the Δgini floor (100.2%) and the Sybil-resistance the live-wealth factor delivers, at zero proof cost and zero incremental leakage. **D2 → Option B.**
- **But the reframe is rule-specific, not automatic.** The naive age/hop provenance signals — the "obvious" value-free factors — are *free to manufacture* and collapse to a *regressive* lottery under an adversary. Only the ADR-0007 construction (public epoch-pool wealth + circulation-gated decay + floor) escapes the Path-C trap, because only circulation-gating makes the favorable signal un-fakeable for free.
- **With `EpochOrigin`, the last hard CT gadget dissolves.** Combined with the already-solved demurrage inequality (§2 of `ct-economics-gadgets.md`) and the dissolved tag-blend (§5 there) and lottery (Path C), **the CT economics reconcile with no ZK beyond ADR-0006 range proofs.** Option A stands only as the fallback for a maintainer who rejects the bounded drip-mine residual — the same residual ADR 0007 already ratified for the bridge.
