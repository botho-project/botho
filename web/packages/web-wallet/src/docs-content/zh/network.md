## 网络信息

本页提供有关 Botho 网络的技术细节，包括连接信息、网络参数和安全模型。

### 网络状态

Botho 网络目前处于 **测试网** 阶段。这意味着：

- 币没有货币价值
- 网络在开发过程中可能会被重置
- 功能仍在测试和完善中
- 欢迎提交错误报告和反馈

生产主网上线时将会公布。

### 连接到网络

**种子发现：**

引导对等节点通过 DNS TXT 记录发现，而非硬编码列表：

| 网络 | DNS 种子域名 |
|---------|-----------------|
| Mainnet | seeds.botho.io |
| Testnet | seeds.testnet.botho.io |

当你的节点启动时，它会解析种子域名以获知引导对等节点（你也可以在配置中固定明确的 `bootstrap_peers`）。初次发现之后，你的节点会与多个对等节点保持连接以实现冗余。

**对等节点发现：**

Botho 使用 libp2p 进行联网，它支持多种发现机制：

- **引导节点** - 用于初次连接的已知种子节点
- **mDNS** - 用于开发的本地网络发现
- **Kademlia DHT** - 分布式对等节点发现
- **Gossipsub** - 基于主题的消息传播

### 网络参数

**区块生产：**

| 参数 | 取值 | 说明 |
|-----------|-------|-------------|
| 出块时间 | 3–40 秒（负载自适应） | 高流量时出块快（仅在 20+ tx/s 时为 3 s），空闲时出块慢 |
| 最大区块大小 | 20 MB | 最大序列化区块大小 |
| 每区块最大交易数 | 5,000 | 交易数量上限 |

**交易限制：**

| 参数 | 取值 | 说明 |
|-----------|-------|-------------|
| 最大输入数 | 16 | 每笔交易的最大输入数 |
| 最大输出数 | 16 | 每笔交易的最大输出数 |
| 环大小 | 20 | CLSAG 环签名中的成员数 |
| 最大交易大小 | 100 KB | 最大序列化交易大小 |

**手续费：**

| 参数 | 取值 | 说明 |
|-----------|-------|-------------|
| 手续费公式 | per-byte rate × size × cluster factor × output penalty | 基于大小、随财富渐进 |
| 集群系数 | 1x–6x | 源自币来源的渐进式乘数 |
| 输出惩罚 | quadratic, capped at 100x | 反 UTXO 农场 |
| 手续费去向 | 80% lottery / 20% burned | 再分配加上通缩压力 |

### 端口参考

默认值因网络而异（主网 / 测试网）；所有端口均可配置：

| 端口（mainnet / testnet） | 协议 | 用途 |
|--------------------------|----------|---------|
| 7100 / 17100 | TCP | P2P gossip (libp2p) |
| 7101 / 17101 | HTTP + WebSocket | JSON-RPC API 与实时更新 |
| 9090 / 19090 | HTTP | Prometheus 指标 |

### 网络安全

**抗女巫攻击（Sybil）：**

网络通过以下方式抵御女巫攻击：
- 基于法定人数的共识（SCP）
- 对等节点的信誉评分
- 区块铸造的资源要求

**日蚀（Eclipse）防护：**

节点通过以下方式防范日蚀攻击：
- 维护多样化的对等节点连接
- 优先选择有既往历史的对等节点
- 定期轮换对等节点
- 多种独立的对等节点发现方法
