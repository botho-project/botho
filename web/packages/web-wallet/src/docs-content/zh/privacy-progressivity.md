## Botho 如何同时实现隐私与渐进性

这是 Botho 最出人意料的成就：**隐私与渐进式经济机制协同运作，而非相互对立。**

传统观点认为这两个目标不可兼得。要对富人征更多的税，你就得知道谁是富人。但知道谁是富人就意味着追踪身份和余额——这与隐私恰恰相反。

Botho 证明了这种取舍是错误的。下面说明其原理。

### 表面上的不可能

想一想“渐进式征税”传统上需要什么：

1. **识别纳税人** —— 将交易与个人关联
2. **追踪其财富** —— 知道他们拥有多少
3. **应用累进税率** —— 对拥有更多者收取更多

每一步都侵犯隐私。如果你知道 Alice 拥有 1M 枚币，你就已经摧毁了她的财务隐私。

这造成了一个看似根本性的冲突：

| 目标 | 需要 |
|------|----------|
| **隐私** | 隐藏谁拥有什么 |
| **渐进性** | 知道谁拥有什么 |

大多数项目接受了这种取舍，只能二选一：
- **完全隐私**（Monero、Zcash）：无法实现渐进式手续费
- **完全透明**（Bitcoin、Ethereum）：可以实现渐进式手续费，但没有隐私

Botho 走了第三条路。

### 关键洞见：来源，而非身份

突破在于认识到：渐进式征税并*不要求*知道身份或总财富。它要求的是**将手续费与经济行为相关联**。

Botho 不问“谁拥有这枚币，他们有多富有？”，而是问：

> “这枚币从何而来，它流通了多少？”

这个问题有一个出人意料的特性：**它可以在链上回答，而无需与身份关联。**

### 我们追踪的是：铸造邻近度

Botho 中的每一枚币都携带一个“集群标签”——记录着是哪次铸造事件创造了它。这追踪的不是*所有者*，而是*来源*。

| 币的状态 | 标签 | 手续费水平 |
|------------|------|-----------|
| 刚刚铸造 | `{cluster_A: 100%}` | 高 |
| 充分流通 | `{cluster_A: 5%, cluster_B: 15%, ...}` | 低 |

关键特性：

| 特性 | 效果 |
|----------|--------|
| **集中的标签** | 近期铸造、尚未流通 → 手续费高 |
| **多样化的标签** | 经多手交易 → 手续费低 |
| **抗拆分** | 拆分会保留标签的集中度 |
| **抗衰减** | 只有真实的商业活动才能降低标签 |

### 为什么这是渐进的

这里开始变得有趣。**铸造邻近度以可预测的方式与财富集中度相关联**：

**新的铸造者往往较为富有。** 赚取区块奖励需要硬件、电力和可靠的在线时间。即便采用 CPU 平等主义的 RandomX 挖矿，新铸的币也不成比例地流向已有资源的人。

**活跃的交易者往往是商家。** 小企业和普通用户频繁交易，使其币与来自众多来源的币混合。

**囤积者保持集中的标签。** 如果你挖到币后一直持有而不交易，你的标签永远不会衰减。你会持续支付高额手续费。

**商业活动自然使标签多样化。** 每一笔合法交易都会将你的标签与交易对手的标签混合。经济活动会自动降低你的手续费率。

这造就了一种**行为上的相关性**：

| 行为模式 | 标签状态 | 手续费水平 |
|------------------|-----------|-----------|
| 挖矿并持有（富人行为） | 集中 | 高 |
| 活跃商业活动（商家行为） | 多样化 | 低 |
| 普通用户消费 | 混合 | 中→低 |
| 巨鲸累积 | 集中 | 高 |

我们不需要知道你*是谁*。我们只需观察*你的币如何流动*。

### 何者保持私密

这一点至关重要：**集群标签揭示的是来源，而非身份。**

| 信息 | 状态 |
|-------------|--------|
| 谁拥有某个 UTXO | **私密**（环签名） |
| 谁收到了付款 | **私密**（隐匿地址） |
| 转账金额 | 在目标设计中为**私密**（保密交易——正在开发中，ADR 0006；在当前测试网上为公开） |
| 你的总余额 | **私密**（无账户关联） |
| 你花费了哪个 UTXO | **私密**（环签名） |
| 币源自何处 | 公开（集群标签） |
| 标签的多样化程度 | 公开（用于手续费计算） |

你揭示了*某些信息*——币的历史——但不会揭示*你是谁*或*你拥有什么*。

### 与环签名的整合

环签名与集群标签无缝协作：

**环签名如何运作：** 当你花费时，你证明“我拥有这 20 个 UTXO 中的某一个”，而不透露是哪一个。这提供了发送方隐私。

**挑战所在：** 如果我们不知道你花费的是哪个 UTXO，我们如何计算正确的手续费？

**解决方案：基于质心的验证。**

1. 全部 20 个环成员的标签都是公开可见的
2. 手续费源自环内标签的**价值加权质心**，并设有下限，使廉价的背景诱饵无法拉低该系数
3. 你声明的输出标签必须与该质心至少有 70% 的相似度，否则验证者会拒绝该交易

```
环成员的标签 → 价值加权质心 → 集群系数
声明的输出标签：必须与质心 ≥ 70% 相似
```

这意味着**隐私不会促成手续费逃避。** 一个大额的真实输入会主导其所在环的质心，因此挑选低系数的诱饵会产生一个不可信的环，从而未通过验证，而不是获得折扣。

### 抽奖再分配

手续费通过集群倾斜的抽奖回流至社区：

**所有手续费的 80%** 被再分配给符合条件的 UTXO。**20% 被销毁**（通缩性）。

**选择机制如何运作：**

```
权重 = 价值 ÷ 集群系数

充分流通的 UTXO（系数 1x）：  每 BTH 获得全额权重
巨鲸集群的 UTXO（系数 6x）：  每 BTH 获得 1/6 的权重
```

**充分流通的币每 BTH 中奖概率比集中财富高出多达 6 倍。** 这是在不知道任何人身份的情况下实现的渐进式再分配——而且由于权重基于价值，将一份头寸拆分成多个 UTXO 绝不会增加其总权重。

**资格门槛**防止博弈：一个 UTXO 必须至少存续 720 个区块，且价值至少为 1 µBTH 才能参与。

### 抗攻击性

我们对这些机制进行了广泛测试：

**拆分攻击：**
```
攻击者：将 1M BTH 拆分为 1000 × 1K BTH
结果：每一份仍带有 {whale_cluster: 100%}
手续费降低：0%（集群系数不变）
结论：攻击被挫败
```

**女巫攻击（多账户）：**
```
攻击者：创建 100 个女巫地址，向每个发送 10K
结果：每个女巫的 UTXO 继承巨鲸标签
手续费降低：极小（标签会传播）
结论：攻击被挫败
```

**滞留攻击（拆分并等待）：**
```
攻击者：拆分成 100 个 UTXO，等待抽奖中奖
结果：权重基于价值——拆分不会获得任何权重
        集群系数会被继承——倾斜仍然对你不利
        二次方输出手续费使拆分本身的成本高达 100 倍
结论：攻击被挫败
```

**对敲交易（自我转账）：**
```
攻击者：快速向自己转账以衰减标签
结果：存续时间门槛要求每次衰减需 720 个区块
        每天最多约 12 次衰减 ≈ 每日衰减 46%
        1 周的对敲交易：约 99% 衰减
        成本：约 84 笔交易手续费（每笔都注入抽奖）
结论：昂贵、缓慢、可被检测
```

### 完整图景

Botho 通过分层设计实现隐私 + 渐进性：

| 层级 | 隐私特性 | 渐进特性 |
|-------|-----------------|---------------------|
| **发送方** | 环签名（20 选 1） | 质心验证的标签传播 |
| **收款人** | 隐匿地址 | — |
| **金额** | Pedersen 承诺（正在开发中——ADR 0006） | — |
| **手续费率** | — | 集群系数曲线（1-6×） |
| **持有成本** | — | 对闲置的富人集群币征收滞留费 |
| **再分配** | 可验证的随机抽取 | 集群倾斜的权重（价值 ÷ 系数） |

每一层都对两个目标都有贡献，互不冲突。

### 为什么这很重要

这不仅仅是一项技术成就——它改变了可能性的边界：

**对用户而言：** 你既获得财务隐私，又获得公平的经济体系。无需取舍。

**对商家而言：** 低手续费奖励经济活动。你交易得越多，费率就越低。

**对网络而言：** 财富自然地从集中的持有流向分散的持有。自我托管相较于托管服务受到奖励。

**对生态系统而言：** 我们证明了“隐私对公平”是一个伪二分法。其他项目也可以采用这些技术。

### 小结

Botho 的隐私 + 渐进性机制之所以有效，是因为：

1. **我们追踪来源，而非身份** —— 币从何而来，而非谁拥有它们
2. **来源与财富行为相关联** —— 新的铸造者较为富有，活跃的交易者是商家
3. **环签名保护隐私** —— 质心验证防止博弈
4. **集群倾斜的抽奖是渐进的** —— 充分流通的币每 BTH 中奖更多
5. **基于价值的权重与二次方输出手续费遏制攻击** —— 滞留与拆分无利可图
6. **商业活动受到奖励** —— 标签通过合法交易衰减；闲置的集中财富支付滞留费

结果是：**首个让你既能保持私密、又能公平地为网络做出贡献的加密货币，在这里，富人支付更多，却无人知道他们是谁。**
