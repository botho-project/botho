## 代币经济

Botho（BTH）采用为长期可持续性而设计的两阶段发行模型：先是带有减半的初始分发阶段，随后是以稳定通胀为目标的永久尾部发行。

### 概览

| 参数 | 取值 |
|-----------|-------|
| 代币符号 | BTH |
| 最小单位 | picocredit (10⁻¹² BTH) |
| 预挖 | 无（100% 挖矿产出） |
| 第一阶段供应量 | ~611 million BTH |
| 出块时间 | 3–40 秒（负载自适应；5 秒货币基线） |

### 单位系统

BTH 采用 12 位小数精度。picocredit 是唯一的基础单位——链上传输的每个金额（余额、手续费、集群财富）都以 picocredits 计价，只有在显示层才会格式化为 BTH：

- **1 picocredit** = 0.000000000001 BTH（最小单位）
- **1 microBTH (µBTH)** = 1,000,000 picocredits = 0.000001 BTH
- **1 milliBTH (mBTH)** = 1,000,000,000 picocredits = 0.001 BTH
- **1 BTH** = 1,000,000,000,000 picocredits

## 发行计划

所有货币计算均假设 5 秒的高负载出块时间（实际出块区间为 3–40 s，仅在极高负载即 20+ tx/s 时降至 3 s）。当网络空闲、出块变慢（最长 40 s）时，发行会按比例拉长——这是一种自然的阻尼器：繁忙的网络按完整计划发行，空闲的网络则发行更少。

### 第一阶段：减半期（满负载下约 5 年）

铸造奖励从 50 BTH 起，每 **6,307,200 个区块**（按 5 秒出块计为一年）减半一次。经过五个减半纪元后，第一阶段已分发 **611,010,000 BTH**：

| 纪元 | 铸造奖励 | 累计供应量 |
|-------|----------------|-------------------|
| 1 | 50 BTH | ~315.4M BTH |
| 2 | 25 BTH | ~473.0M BTH |
| 3 | 12.5 BTH | ~552.0M BTH |
| 4 | 6.25 BTH | ~591.3M BTH |
| 5 | 3.125 BTH | ~611.0M BTH |

（这是在 issue #351 中批准的规范计划，由节点中的回归测试锁定。）

### 第二阶段：尾部发行

第一阶段之后，Botho 过渡到以 **2% 年度净通胀**（总发行减去手续费销毁）为目标的永久尾部发行，并通过难度调整来达成该目标。

**为什么要有尾部发行？**

- **安全预算** - 确保铸造者始终有激励去保护网络
- **丢失币的补充** - 补偿因遗忘密钥而丢失的币
- **可预测的货币政策** - 2% 低于典型法币的通胀

在约 611M BTH 的第一阶段供应量和满负载下，2% 大致相当于每个区块 **约 2 BTH**，并随供应量缓慢增长。

## 手续费结构

### 交易手续费

每笔手续费按两部分拆分：**80% 通过集群倾斜的抽奖再分配**给持有者，**20% 被销毁**，产生抵消尾部发行的通缩压力。

```
fee = per-byte rate × transaction size × cluster factor × output penalty + memo fees
```

| 参数 | 取值 |
|-----------|-------|
| 手续费依据 | 交易大小（字节），而非金额 |
| 集群系数 | 1x–6x 渐进式乘数 |
| 输出惩罚 | 输出数量的二次方，上限 100x |
| 备注手续费 | 每条加密备注固定收费 |
| 手续费去向 | 80% 进入抽奖池 / 20% 销毁 |
| 优先级 | 手续费越高 = 确认越快 |

### 基于集群的渐进式手续费

Botho 实现了一套新颖的 **渐进式手续费系统**，在不为女巫攻击提供可乘之机的前提下对财富集中征税。

**核心创新：** 手续费不基于交易金额（通过拆分很容易被规避），而是基于币的*来源*——即币最初来自哪里。

| 参数 | 取值 |
|-----------|-------|
| 集群系数范围 | 1x 至 6x 乘数 |
| 曲线形状 | 在 log(集群财富) 上的 S 形曲线，在 100K BTH 处中点为 3.5x |
| 标签衰减 | 每次符合条件的跳转衰减 5% |

**为什么它能抗女巫攻击：** 拆分币并不会改变它们的来源。无论巨鲸的币是存放在 1 个 UTXO 还是 1000 个 UTXO 中，它们都带有相同的集群标签。

### 滞留费

交易手续费是一种消费税——它无法触及从不流动的财富。滞留费弥补了这一空白：一种 **针对富有集群币的持有费，在它们最终被花费时支付**。

| 参数 | 取值 |
|-----------|-------|
| 费率 | 在最高（6x）集群系数下，每年 2% |
| 缩放 | 与 (factor − 1) 成正比——系数为 1 的币支付 **零** 费用 |
| 引导期 | 在第一个减半纪元内禁用 |
| 所得 | 流入抽奖重新分配池 |

日常使用的币从不支付滞留费；它只约束集中且闲置的财富。频繁倒手也无法逃避它——花费时会先支付累积的费用，因此在任何持有期内支付的总额都相同，无论你自我转账多少次（而且每次转账还会额外产生手续费）。

## 经济设计理念

### 为什么没有预挖？

- **公平分发** - 每个人起点平等；早期铸造者承担风险
- **可信度** - 没有内部人优势或创始人牟利
- **去中心化** - 从第一天起就没有集中持有

### 为什么将手续费按 80/20 拆分？

- **再分配** - 80% 通过集群倾斜的抽奖回流给持有者，偏向充分流通的币
- **通缩压力** - 20% 的销毁抵消尾部发行
- **可预测** - 净通胀 = 总发行 − 销毁

### 为什么采用渐进式集群手续费？

- **降低集中度** — 富有集群支付更多
- **抗女巫攻击** — 无法通过拆分账户来规避
- **鼓励流通** — 移动币会扩散标签，从而降低手续费
- **兼容隐私** — 可与环签名和隐匿地址协同工作
