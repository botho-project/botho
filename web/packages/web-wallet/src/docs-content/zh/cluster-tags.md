## 集群标签

集群标签是 Botho 用于追踪币的来源而不损害隐私的新颖机制。它们实现了**抗女巫的渐进式手续费**、**基于抽奖的重新分配**以及**保护隐私的环签名**。

### 问题：无身份的财富征税

传统的渐进式征税需要身份。在加密货币中：

- **基于金额的手续费**会瞬间失效——将 100 万拆分成 1000 × 1000 并支付更低的费率即可
- **基于账户的征税**不可能实现——任何人都可以创建无限多的地址
- **交易计数**行不通——机器人可以制造虚假活动

**核心挑战：** 当你无法识别谁拥有什么时，如何对财富集中征税？

### 解决方案：来源追踪

我们追踪的不是*谁*拥有币，而是它们*来自哪里*。每一枚币都携带着对其起源的记忆。

**关键洞见：** 拆分币并不会改变它们的来源。

### 集群标签如何运作

**1. 集群诞生于铸造之时**

每个区块奖励都会创建一个唯一的“集群”——即由某个特定铸造者所铸造的币的身份标识。该铸造奖励为那个集群携带一个 100% 的标签。

```
Block 1000: Minter A receives 50 BTH
  → Tag: {cluster_A: 100%}

Block 1001: Minter B receives 50 BTH
  → Tag: {cluster_B: 100%}
```

**2. 标签在转移时继承**

当币发生移动时，收款人的 UTXO 会继承发送方的标签：

```
Minter A → Merchant → Customer
  100%   →   95%    →   90%   (A's cluster factor)
```

**3. 标签在合并时混合**

当多个输入被一起花费时，输出标签是按价值加权的平均值：

```
Input 1: 70 BTH {cluster_A: 100%}
Input 2: 30 BTH {cluster_B: 100%}
─────────────────────────────────
Output: 100 BTH {cluster_A: 70%, cluster_B: 30%}
```

**4. 标签随时间衰减**

每一次交易跳转都会使标签衰减 5%，将归属分散到整个经济体中。但衰减只有在 UTXO 至少已存在 720 个区块时才会生效（约一到几个小时，取决于负载自适应的出块时间），从而防止对敲交易攻击。

### 为什么拆分行不通

这正是集群标签的特别之处：

```
Whale splits 1,000,000 BTH into 1000 × 1000 BTH

Before: 1 UTXO with {whale_cluster: 100%}
After:  1000 UTXOs, each with {whale_cluster: 100%}

Fee rate: unchanged (based on cluster wealth, not UTXO count)
```

一个集群的“源头财富”是该铸造者铸造的总价值——拆分并不会减少它。

### 渐进式手续费曲线

集群因子决定你需要支付多少。它遵循一条关于集群财富*对数*的平滑 S 形曲线，其中点位于 100,000 BTH：

| 集群财富 | 手续费倍数 |
|----------------|----------------|
| 小型集群（≤ 约 1K BTH） | 约 1x（基础费率） |
| 中型集群（约 100K BTH） | 约 3.5x（曲线中点） |
| 巨鲸集群（≥ 约 10M BTH） | 约 6x（饱和） |

该倍数应用于基于大小的手续费（`per-byte rate × transaction size`），因此同一笔转移对巨鲸集群而言最多要花费充分流通的币的 6 倍——而且无论如何拆分都无法改变这一点。

### 基于抽奖的重新分配

所有交易手续费的 80% 通过抽奖重新分配给符合条件的 UTXO。20% 被销毁。

**运作方式：**

1. 每笔交易根据集群因子支付一笔手续费
2. 手续费的 80% 在以可验证随机性抽取的 4 名中奖者之间分配
3. 20% 被永久销毁（通缩）

**中奖者如何选出（集群加权）：**

一个 UTXO 的中奖权重是它的**价值除以其集群因子**。这是唯一一种拆分不变的渐进式加权方式：

- 权重基于价值，因此将一个头寸拆分成多个 UTXO 绝不会增加总权重
- 倾斜来自集群来源，它会通过拆分而继承
- 充分流通（低因子）的币按比例中奖更多；巨鲸集群的币中奖更少

要参与抽奖，UTXO 必须至少已存在 720 个区块，且价值至少为 1 µBTH。

### 环签名与标签隐私

集群标签与 CLSAG 环签名无缝协作：

**挑战：** 环签名在 20 个环成员中隐藏了哪个输入是真实的。我们如何计算正确的手续费？

**解决方案：** 基于质心的验证

1. 所有环成员的标签都是公开已知的
2. 手续费由环内标签的按价值加权*质心*得出，并设有下限以阻止廉价的背景诱饵把因子拉低
3. 所声明的输出标签必须与环质心至少有 70% 的相似度（余弦相似度），否则交易将被拒绝

这可以防止手续费规避——你无法精心挑选低因子的诱饵来削减你的手续费，因为不合理的环组合会通不过验证。

### 衰减机制细节

为防止对敲交易（反复向自己转账以衰减标签）：

**基于账龄的门控：**
- 衰减只应用于至少已存在 720 个区块的 UTXO
- 快速自我转账产生的新输出不会衰减
- 账龄门控自然地将衰减上限限定在每天约 12 次

**天然的速率限制：**

| 攻击 | 结果 |
|--------|--------|
| 100 次快速自我转账 | 0% 衰减（所有输出都太新） |
| 耐心攻击（1 天） | 最多约 46% 衰减（仅约 12 次符合条件的跳转） |
| 耐心攻击（1 周） | 约 99% 衰减——但你已支付了约 84 笔交易手续费 |
| 持有而不交易 | 0% 衰减 |

### 隐私方面的考量

**第 1 阶段（当前）：** 标签在 UTXO 上是公开的。这使得手续费可以直接验证，但会泄露一些来源信息。

**第 2 阶段（计划中）：** 标签将使用带零知识证明的 Pedersen 承诺来隐藏。验证者在看不到实际标签值的情况下验证手续费是否正确。

### 经济激励

集群标签系统创造了一致的激励：

| 行为 | 对标签的影响 | 激励 |
|----------|----------------|-----------|
| **让币流通** | 标签衰减并混合 | 更低的手续费，更高的抽奖权重 |
| **囤积财富** | 标签保持集中 | 更高的手续费，更低的抽奖权重，滞留费 |
| **拆分成许多 UTXO** | 标签不变 | 没有好处——手续费和抽奖权重都基于来源和价值 |

### 技术参数

| 参数 | 值 | 用途 |
|-----------|-------|---------|
| 衰减率 | 每次符合条件的跳转 5% | 标签逐步扩散 |
| 最小 UTXO 账龄（衰减 + 抽奖） | 720 blocks | 防止对敲交易 |
| 最小 UTXO 价值（抽奖） | 1 µBTH | 排除尘埃 |
| 集群因子范围 | 1x–6x（中点为 100K BTH 处的 3.5x） | 渐进式手续费 |
| 环大小 | 20 | 标签传播的隐私集 |
| 抽奖中奖者 | 每次抽取 4 名 | 重新分配的粒度 |
| 销毁率 | 手续费的 20% | 通缩压力 |
| 奖池率 | 手续费的 80% | 重新分配的数额 |
| 滞留费 | 最大因子下每年 2%（见“代币经济”） | 触及闲置的集中财富 |

### 小结

集群标签解决了困扰加密货币的“抗女巫渐进式征税”难题：

1. **追踪来源，而非身份** — 币会记住自己的起源
2. **抵御拆分攻击** — 集群财富在铸造时即已固定
3. **实现渐进式手续费** — 富有的集群支付更多
4. **驱动公平的重新分配** — 抽奖权重向充分流通的币倾斜
5. **保护隐私** — 与环签名协同工作
6. **鼓励流通** — 标签通过商业活动衰减

这使 Botho 成为第一个拥有可信机制、能够实现难以被轻易规避的基于财富的手续费的加密货币。
